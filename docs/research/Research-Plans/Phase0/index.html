Phase0

📅 Phase 0: Before Enter (2025/10 - 2026/02)​

目标：完成Demo RIS + 射频入门

短板（诚实面对）​

❌ 射频硬件：几乎空白
   ├─ CST/HFSS：没用过
   ├─ VNA：没碰过
   ├─ 射频PCB：只是乱画过
   └─ 电路分析：三极管看不懂

⚠️ 硬件基础：浅层
   ├─ PCB：会画但不深入
   ├─ 模拟电路：weak
   ├─ 数字电路：大四摆烂没学好
   └─ 嵌入式：只会用模组，不懂底层

⚠️ FPGA：接触过但不熟
   ├─ Verilog写过
   ├─ 工具链不熟
   └─ 需要系统学习

❌ OS底层：用户层面
   ├─ 没看过内核代码
   ├─ 没写过内核模块
   └─ 需要从零开始

⚠️ 数学：够用但浅
   ├─ 被高中物竞cover
   ├─ 大学数学浅
   └─ 可能需要补充

Month 1-2 (Oct-Nov 2025): 射频理论突击​

目标：从零到能看懂射频设计

Week 1-3: 电磁场和射频基础​

补课（必须！）：

Week 1: 复习电磁场理论 （2-3天）
├─ 复习麦克斯韦方程（物竞学过）
├─ 波动方程
├─ 边界条件
├─ 反射、折射、衍射
└─ 重点：理解物理本质
    └─ 有物竞底子，这部分应该快

书籍：
├─ 《电磁场与电磁波》（谢处方）- 快速过
├─ 或《电磁场理论基础》- 选读
└─ 重点：理解概念，不死抠数学

每天时间：4-6小时
产出：建立电磁场框架

Week 1-3: 射频电路基础
├─ 传输线理论
│   ├─ 特性阻抗
│   ├─ 反射系数
│   ├─ 驻波比
│   └─ Smith圆图
│
├─ S参数
│   ├─ S11/S21意义
│   ├─ 如何测量
│   ├─ 如何优化
│   └─ 实例分析
│
├─ 阻抗匹配
│   ├─ 为什么要匹配
│   ├─ L型/π型匹配
│   ├─ 微带线匹配
│   └─ 仿真验证
│
└─ 天线基础
    ├─ 方向图
    ├─ 增益
    ├─ 极化
    └─ 天线类型

书籍/资源：
├─ 《射频电路设计》（Ludwig）- 前5章
├─ YouTube: "RF Basics" playlist
├─ B站：射频工程师养成记
└─ 每天看视频2小时 + 读书2小时

补模拟电路（必须！）：
├─ 三极管/MOS管基本原理
├─ 放大电路
├─ 振荡电路
└─ 目标：能看懂射频电路图

每天时间：6-8小时
产出：
✅ 理解射频基本概念
✅ 看懂射频论文的硬件部分
✅ 能用Smith圆图

Week 4-7: 仿真软件实战​

CST Studio Suite（主攻）：

Week 4-6: CST基础
├─ 安装和界面（1天）
├─ 官方Tutorial（2天）
│   ├─ 跑完所有基础例程
│   ├─ 理解workflow
│   └─ 熟悉操作
│
├─ 第一个设计：Patch天线（3天）
│   ├─ 2.4GHz微带天线
│   ├─ 计算尺寸（公式）
│   ├─ 建模仿真
│   ├─ 参数扫描
│   └─ 优化到S11<-10dB
│
└─ 理解仿真结果（3天）
    ├─ S参数的意义
    ├─ 远场方向图
    ├─ 输入阻抗
    └─ 表面电流分布

Week 6-7: RIS单元初探
├─ 文献学习（3天）
│   ├─ 找5篇有硬件的RIS论文
│   ├─ 重点看设计部分
│   ├─ 理解相位调制原理
│   └─ 提取设计思路
│
├─ 简单RIS单元仿真（7天）
│   ├─ 选择最简单的结构
│   │   └─ PIN二极管 + 微带结构
│   │
│   ├─ CST建模
│   │   ├─ 基本单元
│   │   ├─ PIN管导通/关断两个状态
│   │   └─ 仿真相位差
│   │
│   ├─ 参数优化
│   │   ├─ 调整尺寸
│   │   ├─ 优化相位差
│   │   └─ 降低损耗
│   │
│   └─ 理解原理
│       ├─ 为什么能调相位
│       ├─ 损耗来源
│       └─ 如何改进
│
└─ 设计文档
    ├─ 记录设计过程
    ├─ 参数和结果
    ├─ 问题和思考
    └─ 为实际制作准备

每天时间：8-12小时（全力投入）
产出：
✅ CST熟练使用
✅ 完成第一个RIS单元仿真
✅ 理解RIS工作原理
✅ 建立信心！

Month 3-4 (Dec 2025 - Jan 2026): Demo RIS制作​

目标：4单元Demo RIS + 初步测试

Week 8-9: PCB设计​

设计目标：
├─ 4个RIS单元（够demo）
├─ 工作频率：2.4GHz（WiFi频段，方便测试）
├─ 1bit相位控制（0°/180°，最简单）
├─ 手动控制（先不联网，降低复杂度）
└─ 能工作即可，不追求性能

详细步骤：

Day 1-3: 原理图设计
├─ RIS单元电路
│   ├─ 从仿真转化为实际电路
│   ├─ PIN二极管选型（BAP64-02）
│   ├─ 偏置电路设计
│   ├─ 控制接口（GPIO）
│   └─ 电源设计（3.3V/5V）
│
├─ 控制电路
│   ├─ ESP32模组（熟悉）
│   ├─ GPIO扩展
│   ├─ 电源管理
│   └─ 调试接口
│
└─ 原理图检查
    ├─ DRC检查
    ├─ 网表检查
    └─ 同学/老师review

Day 4-7: PCB布局（关键！）
├─ Layer stackup设计
│   ├─ 4层板（必须）
│   ├─ TOP: 信号+RIS单元
│   ├─ L2: GND（完整平面）
│   ├─ L3: Power
│   └─ BOTTOM: 信号+控制
│
├─ RIS单元布局
│   ├─ 精确尺寸（按照仿真）
│   ├─ 单元间距
│   ├─ 接地设计
│   └─ 馈电位置
│
├─ 射频走线规则
│   ├─ 50Ω阻抗控制
│   │   └─ 用Saturn计算走线宽度
│   ├─ 地平面完整
│   ├─ 过孔最少
│   ├─ 走线长度匹配
│   └─ 远离干扰源
│
├─ 控制部分布局
│   ├─ 与射频隔离
│   ├─ 电源去耦
│   ├─ 走线规范
│   └─ 调试方便
│
└─ PCB检查
    ├─ DRC（设计规则检查）
    ├─ 阻抗仿真（如果可能）
    ├─ 3D查看
    └─ 最后确认

Day 8-10: 制板和采购
├─ 选择PCB厂商
│   ├─ 嘉立创（推荐）
│   ├─ 4层射频板
│   ├─ 指定板材（RF-35/4350）
│   ├─ 阻抗控制
│   └─ 成本：500-800元
│
├─ 元器件采购
│   ├─ 淘宝/立创商城
│   ├─ PIN二极管
│   ├─ 电阻电容
│   ├─ ESP32模组
│   ├─ SMA接头
│   └─ 成本：300-500元
│
└─ 等待期间
    ├─ 学习焊接技巧（视频）
    ├─ 准备测试设备
    └─ 设计测试方案

成本：1000-1500元

Week 10-11: 组装和调试​

Week 10: 焊接和组装

Day 1-2: PCB到货和检查
├─ 外观检查
├─ 测量关键尺寸
├─ 通断测试
└─ 阻抗测试（如果有设备）

Day 3-5: 焊接
├─ 控制部分先焊（容易）
│   ├─ 电源部分
│   ├─ ESP32模组
│   ├─ GPIO电路
│   └─ 调试接口
│
├─ 测试控制部分
│   ├─ 上电测试
│   ├─ ESP32能跑
│   ├─ GPIO能控制
│   └─ 确认OK
│
├─ RIS单元焊接（小心）
│   ├─ 贴片元件（手工或热风枪）
│   ├─ PIN二极管
│   ├─ 偏置电路
│   └─ SMA接头
│
└─ 外观检查
    ├─ 焊点
    ├─ 桥接
    ├─ 冷焊
    └─ 修复问题

Day 6-7: 基本电气测试
├─ 上电测试
│   ├─ 电压正常
│   ├─ 电流合理
│   ├─ 无短路
│   └─ 各部分工作
│
├─ 控制测试
│   ├─ GPIO能控制PIN管
│   ├─ 状态切换正常
│   ├─ 稳定性
│   └─ 简单程序测试
│
└─ 问题记录
    └─ 发现的所有问题

Week 11: 射频测试（关键！）

需要设备（借用or租用）：
├─ VNA（矢量网络分析仪）
│   └─ 实验室可能有
│   └─ 或租用（500-1000元/天）
│
├─ 天线和线缆
├─ 功率计（可选）
└─ 频谱仪（可选）

Day 1-3: S参数测试
├─ VNA校准（重要！）
├─ 测试每个单元的S11
│   ├─ PIN导通状态
│   ├─ PIN关断状态
│   ├─ 对比仿真结果
│   └─ 记录数据
│
├─ 测试S21（如果可能）
│   ├─ 发射-RIS-接收
│   ├─ 测量反射/透射
│   └─ 验证工作原理
│
└─ 分析结果
    ├─ 与仿真对比
    ├─ 找出差异原因
    ├─ 改进方向
    └─ 是否需要改版

Day 4-7: 系统测试
├─ WiFi信号测试
│   ├─ 发射端：路由器
│   ├─ RIS：板子
│   ├─ 接收端：ESP32/手机
│   └─ 测量RSSI变化
│
├─ 相位控制验证
│   ├─ 手动切换状态
│   ├─ 观察信号变化
│   ├─ 不同配置对比
│   └─ 验证可控性
│
├─ 拍照和视频
│   ├─ 记录过程
│   ├─ 展示效果
│   └─ 为后续准备材料
│
└─ 写测试报告
    ├─ 详细记录
    ├─ 数据和图表
    ├─ 问题总结
    └─ 改进计划

结果评估：
成功标准（降低期望）：
✅ 能工作（哪怕性能一般）
✅ 能测到相位差异
✅ 能通过控制改变信号
✅ 理解了制作流程

即使不完美也OK：
- 第一次做，必然有问题
- 重要的是学到经验
- 知道哪里需要改进
- 为正式版本铺路

成本：0-2000元（测试设备租用）
